title: "Use all 8 AFS operations: read, write, search, exec"
description: "A deep dive into all 8 AFS operations — read, write, list, delete, search, stat, explain, and exec — with TypeScript examples and return shapes."

AFS exposes every data source through exactly 8 operations. Every provider that implements the `AFSModule` interface speaks the same contract, so the code below works identically whether the path points to a local file, a SQLite table, a Git commit, or a remote API.

All operations return typed result objects. `data` holds the payload; `message` carries optional human-readable context. Errors are thrown as typed `AFSError` subclasses.

  <Tab title="read">

## read

Reads the content and metadata of a single entry. The result's `data` field is an `AFSEntry` containing `content`, `meta`, `path`, `id`, and more.

```typescript
const result = await afs.read("/code/src/index.ts");
// result.data?.content — the file content (string or object)
// result.data?.meta    — provider metadata (size, mimeType, etc.)
// result.data?.path    — canonical path
```

### Options

| Option | Type | Description |
|--------|------|-------------|
| `startLine` | `number` | First line to return (1-indexed, inclusive) |
| `endLine` | `number` | Last line to return (1-indexed, inclusive; `-1` = end of file) |
| `as` | `string` | Request an alternative representation (e.g. `"text"`, `"html"`) |

```typescript
// Read lines 10–20 only
const slice = await afs.read("/logs/app.log", { startLine: 10, endLine: 20 });

// Request a rendered representation
const html = await afs.read("/docs/README.md", { as: "html" });
```

### Alternative representations (`.as/`)

You can also use path syntax to request alternative representations:

```typescript
const text = await afs.read("/document.pdf/.as/text");
```

## write

Creates or updates an entry. Pass a payload with `content`, `meta`, or both.

```typescript
await afs.write("/code/output.txt", { content: "Generated by AFS" });
```

### Payload shape

```typescript
interface AFSWriteEntryPayload {
  content?: string | Record<string, unknown>;
  meta?: Record<string, unknown>;
  patches?: AFSPatch[];
}
```

### Options — write mode

| Mode | Behaviour |
|------|-----------|
| `"replace"` | Default. Overwrite if exists, create if not. |
| `"create"` | Only create new. Fails if entry already exists. |
| `"update"` | Only update existing. Fails if entry does not exist. |
| `"append"` | Append content to the end of an existing entry. |
| `"prepend"` | Prepend content to the beginning. |
| `"patch"` | Apply structured `AFSPatch` operations (str_replace, insert_before, etc.). |

```typescript
// Append a log line
await afs.write("/logs/app.log", { content: "\n2026-04-21 boot" }, { mode: "append" });

// Patch with a targeted string replacement
await afs.write(
  "/code/config.ts",
  {
    patches: [{ op: "str_replace", target: "VERSION = '1.0'", content: "VERSION = '2.0'" }],
  },
  { mode: "patch" },
);
```

### Return shape

```typescript
interface AFSWriteResult {
  data: AFSEntry;   // The written entry (with updated content/meta)
  message?: string;
}
```

## list

Lists the direct children of a path. Returns an array of `AFSEntry` objects.

```typescript
const { data: entries } = await afs.list("/code/src");
for (const entry of entries) {
  console.log(entry.path, entry.meta?.size);
}
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxDepth` | `number` | `1` | How many levels deep to recurse |
| `offset` | `number` | `0` | Pagination: skip this many entries |
| `limit` | `number` | `1000` | Pagination: return at most this many entries |
| `pattern` | `string` | — | Glob pattern to filter entries (e.g. `"**/*.ts"`) |
| `orderBy` | `[string, "asc"\|"desc"][]` | — | Sort by field |

```typescript
// Recursive listing (all descendants)
const { data: all } = await afs.list("/code", { maxDepth: 10 });

// Filter TypeScript files only
const { data: tsFiles } = await afs.list("/code", { pattern: "**/*.ts" });

// Paginated listing
const page2 = await afs.list("/large-dir", { offset: 100, limit: 50 });
```

### Return shape

```typescript
interface AFSListResult {
  data: AFSEntry[];
  total?: number;   // Present when data is a subset of the full result
  message?: string;
}
```

## delete

Removes an entry. Returns a result object on success; throws `AFSNotFoundError` if the path doesn't exist.

```typescript
await afs.delete("/code/tmp/scratch.ts");
```

Pass `{ recursive: true }` to remove a directory and all its children:

```typescript
await afs.delete("/code/tmp", { recursive: true });
```

### Return shape

```typescript
interface AFSDeleteResult {
  message?: string;
}
```

<Warning>
  Delete is irreversible. Providers backed by real storage (filesystem, S3, SQLite) will permanently remove the data. Some providers (e.g. Git) may not support delete at all and will throw `AFSReadonlyError`.
</Warning>

## search

Searches for entries matching a query string. Each provider implements search in the way that makes most sense for its backend — full-text for files, SQL `LIKE` for databases, semantic for vector stores.

```typescript
const { data: matches } = await afs.search("/code", "TODO");
for (const entry of matches) {
  console.log(entry.path, entry.meta?.matchCount);
}
```

### Options

| Option | Type | Description |
|--------|------|-------------|
| `limit` | `number` | Maximum number of results |
| `caseSensitive` | `boolean` | Whether the match is case-sensitive |

```typescript
const results = await afs.search("/code", "deprecated", {
  limit: 20,
  caseSensitive: false,
});
```

### Return shape

```typescript
interface AFSSearchResult {
  data: AFSEntry[];  // Matching entries; meta may include match context
  message?: string;
}
```

## stat

Returns metadata for a path without fetching content. Cheaper than `read` when you only need to check existence, size, or type.

```typescript
const { data } = await afs.stat("/code/src/index.ts");
console.log(data?.meta?.size);      // bytes
console.log(data?.meta?.kind);      // e.g. "afs:node"
console.log(data?.meta?.childrenCount); // undefined = leaf, -1 = unknown, N = count
```

### Return shape

```typescript
interface AFSStatResult {
  data?: Omit<AFSEntry, "content">;  // All AFSEntry fields except content
  message?: string;
}
```

## explain

Returns a human- and LLM-readable description of a path. Providers use `explain` to self-document their structure, available actions, and capabilities. Agents can call `explain` on any path to discover what's there before acting.

```typescript
const explanation = await afs.explain("/sql");
// explanation.content — Markdown description of this provider
// explanation.actions — available actions with input schemas
```

### Use cases

- **Agent discovery**: Call `explain("/")` to get a top-level overview of everything mounted.
- **Tool generation**: The MCP integration calls `explain` to build tool definitions for Claude, Cursor, etc.
- **Debugging**: Understand what a provider exposes before writing code against it.

```typescript
// Get the full root explanation (all mounted providers)
const root = await afs.explain("/");

// Get provider-specific explanation
const gitDocs = await afs.explain("/git");
```

### Options

```typescript
interface AFSExplainOptions {
  format?: "markdown" | "text";
}
```

## exec

Executes a named action on a provider. Actions live at `/.actions/<name>` paths relative to the mount. Use `exec` for side-effecting operations: running a SQL query, toggling a smart-home device, kicking off a build, or patching an AUP tree.

```typescript
// Execute a SQL query through the SQLite provider
const result = await afs.exec("/sql/.actions/query", {
  sql: "SELECT * FROM users WHERE active = 1",
});
console.log(result.success, result.data);
```

### Return shape

```typescript
interface AFSExecResult {
  success: boolean;
  data?: Record<string, unknown>;
  error?: {
    code: string;
    message: string;
    details?: Record<string, unknown>;
  };
  usage?: {
    tokens?: { input: number; output: number; total: number };
    cost?: number;
    durationMs?: number;
  };
}
```

### Action severity

Every action carries a severity level that AFS enforces against the mount's `actionPolicy`:

| Severity | Meaning | Example |
|----------|---------|---------|
| `"ambient"` | Low-risk, reversible | Toggle a light |
| `"boundary"` | Medium-risk, noticeable | Lock a door |
| `"critical"` | High-risk, hard to reverse | Remote-start a car |

| Policy | Allowed severities |
|--------|--------------------|
| `"safe"` | ambient only |
| `"standard"` | ambient + boundary |
| `"full"` | all |

```typescript
// Mount with a restrictive action policy
await afs.mount(vehicleProvider, "/vehicle", { /* options passed at construction */ });
// vehicleProvider must be constructed with actionPolicy: "standard"

// This will throw AFSSeverityError if the unlock action is "critical"
await afs.exec("/vehicle/.actions/unlock", { targetDoor: "driver" });
```

### Streaming exec

For long-running actions, providers can emit incremental chunks via `onChunk`:

```typescript
await afs.exec(
  "/ai/.actions/generate",
  { prompt: "Write a haiku about AFS" },
  {
    onChunk: (chunk) => {
      process.stdout.write(chunk.text ?? "");
    },
  },
);
```

## Entry shape

All operations return or accept `AFSEntry` objects:

```typescript
interface AFSEntry<T = any> {
  id: string;
  path: string;
  createdAt?: Date;
  updatedAt?: Date;
  content?: T;           // File/node content
  meta?: {
    kind?: string;       // e.g. "afs:node", "database:table"
    size?: number;       // Bytes
    childrenCount?: number; // undefined=leaf, -1=unknown, N=count
    description?: string;
    [key: string]: any;
  } | null;
  actions?: ActionSummary[];  // Available actions for this entry
  summary?: string | null;
  linkTo?: string | null;
}
```
