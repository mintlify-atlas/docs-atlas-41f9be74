AFS (Agentic File System) is a virtual filesystem abstraction that gives AI agents a consistent, composable interface to any data source. Inspired by the Unix "everything is a file" philosophy, AFS extends this to **"everything is context"** вҖ” databases, APIs, cloud services, and local files all become paths that agents can `read`, `write`, `search`, and act on.

## Get started

<Steps>
  <Step title="Install the core package and a provider">
    ```bash
    npm install @aigne/afs @aigne/afs-fs
    ```
  </Step>
  <Step title="Create an AFS instance and mount a provider">
    ```typescript
    import { AFS } from "@aigne/afs";
    import { AFSFS } from "@aigne/afs-fs";

    const afs = new AFS();
    await afs.mount(new AFSFS({ localPath: "./my-project" }), "/code");
    ```
  </Step>
  <Step title="Read, list, and search across your data">
    ```typescript
    // Read a file
    const { data } = await afs.read("/code/src/index.ts");

    // List a directory
    const { data: listing } = await afs.list("/code/src");

    // Search for content
    const { data: results } = await afs.search("/code", "TODO");
    ```
  </Step>
  <Step title="Add more providers and compose a unified namespace">
    ```typescript
    import { AFSGit } from "@aigne/afs-git";
    import { SQLiteAFS } from "@aigne/afs-sqlite";

    await afs.mount(new AFSGit({ repoPath: "./repo" }), "/git");
    await afs.mount(new SQLiteAFS({ url: "file:./data.db" }), "/db");

    // All sources under one roof
    await afs.read("/code/README.md");
    await afs.read("/git/main/src/index.ts");
    await afs.exec("/db/.actions/query", { sql: "SELECT * FROM users LIMIT 10" });
    ```
  </Step>
</Steps>
